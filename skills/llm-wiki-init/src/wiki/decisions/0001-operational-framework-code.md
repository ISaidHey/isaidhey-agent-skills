## Context
The vault needed an operational framework. PARA (Projects, Areas, Resources, Archive) was evaluated and rejected — it is designed for personal note retrieval, not shared pipeline operations.

## Decision
**Use CODE (Capture → Organize → Distill → Express) as the operational framework.**

## Consequences
Pipeline semantics fit the intake→analysis→query workflow cleanly. **Re-evaluate if:** vault shifts from shared pipeline to personal knowledge management.

## Invariants
- Do not substitute PARA as the operational model — PARA as a source directory layout (setup option 2) is distinct and supported
